LifeTime Management Console 11.27.0

This is a preliminary version of the document.

New in LifeTime Management Console 11.27.0

  • By using the new feature Deployment Protection Policy, it is now possible to block specific applications from being deployed to selected environments to prevent accidental deployments. Overrides are allowed (publishing via Service Center or Service Studio), but will trigger a warning in the UI and notify the user who set the block and infra admins for visibility. An infrastructure-level setting can be used to define if new apps are blocked by default. (R11CT-3955)
  • Stagings can now be relabeled. (R11CT-4070)

Bug Fixing

  • Fixed an issue where the 'Staging Settings' screen was being skipped when deploying an application containing 'Entities from External Databases' for the first time to the target environment. This skip was leading to errors in the deployment of such applications. (RPM-5761) Application Lifecycle LifeTime
