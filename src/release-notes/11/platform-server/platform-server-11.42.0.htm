Platform Server 11.42.0

This is a preliminary version of the document.

New in Platform Server 11.42.0

  • Added support for SQL Server 2025 and compatibility level 170. This applies to the platform database, as well as external databases. Configuration Tool and Service Center now feature distinct options for SQL Server and Azure SQL to improve setup clarity.

    (R11BCT-443)

  • Downgraded Oracle Managed Driver to version 19.26 due to thread pooling issues with version 23.26.100.

    (R11BCT-462)

  • Service Center's Site Properties page now shows descriptions for each property and includes a notice warning that these are internal settings. Their default values are recommended and should only be changed when instructed by OutSystems Support.

    (R11BRT-1668)

  • It is now possible to use service account tokens with a 24-hour overlap period when rotating authentication tokens for the Performance Monitoring API. When a new token is generated in LifeTime, the previous token remains valid for 24 hours, allowing integrations to be updated without service interruptions. This requires LifeTime 11.29.0 or later.

    (R11CT-4373)

  • Added support for the form-action and report-to Content Security Policy (CSP) directives. The platform now sends the modern Reporting-Endpoints HTTP header, used by report-to, alongside the legacy report-uri directive for CSP violation reporting, providing improved browser compatibility with the latest CSP specification.

    (R11CT-4724)

  • DropdownCustom now follows the ARIA combobox pattern (combobox/listbox/option roles, aria-expanded, aria-selected) when EnableWCAG22 is enabled, ensuring WCAG 2.2 (4.1.2 — Name, Role, Value) compliance for assistive technologies.

    (R11DT-3623)

  • Updated Swagger UI dependencies dompurify (3.4.0) , immutable (3.8.3) , lodash (4.18.1) and swagger-client (3.37.2) to ensure continued security compliance. This update has no runtime impact.

    (R11DT-3830)
  • Fixed an issue where Mobile applications embedded in an iframe would fail to load on Safari (macOS) and all browsers on iOS/iPadOS. (RPM-6421) Application Runtime

Bug Fixing

  • Fixed an issue that caused unnecessary folders in the running directory on frontends to be created for library modules. These folders would not be cleaned up by the automatic cleanup processes.

    (R11BCT-331)

  • Fixed an issue that could trigger a "You are not allowed to upload this Extension. Please contact your OutSystems Administrator." error when doing a solution publish in an environment registered in LifeTime. The issue would occur in very specific scenarios, when the extension already existed in the system but was not associated to any application.

    (R11BCT-392)
  • Fixed an issue that caused unnecessary folders in the running directory on frontends to be created if there were license errors during Publish. These folders would not be cleaned up by the automatic cleanup processes. (RPM-2158) Infrastructure Management OutSystems Services
  • Fix unnecessary FK Index recreation during module publishing. (RPM-2402) Application Lifecycle LifeTime
  • Fixed an issue where BCC recipients did not receive emails when sending through the modern SMTP implementation (OAuth 2.0) (RPM-6750) Application Runtime Interface
  • Removed form-action CSP directive from Service Center and LifeTime applications since it was conflicting OIDC login. A fix to this conflict was introduced that can be enable by a setting. (RPM-6844) Application Lifecycle Service Center
  • Fixed an issue that caused "The operation was canceled" errors to flood Service Center logs (Errors and Integrations tabs) whenever a client cancelled a request to an exposed REST service - for example, by navigating away or closing the browser tab. Now, these cancellations are no longer logged as errors. You can revert to the previous behavior via Factory Configuration setting "Hide Operation Canceled Exception in Logs". (RPM-778) Application Runtime Logic Execution
  • HtmlSanitizer has been upgraded from version 8.0.838 to the latest patched version (9.0.892+). This upgrade resolves CVE-2026-25543 by shipping a patched version of the library as part of the Sanitization System Component. (RPM-6634) Application Runtime
  • Updated MailKit to version 4.16.0. (RPM-6797) Application Runtime
