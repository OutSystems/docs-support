OutSystems Developer Cloud 2025-01-23

Released on Jan 23, 2025

New

  • On the workflow instance detail screen, any workflow activity with an associated timeout will now display this information under the activity detail. (RDLOT-3410)
  • Workflow Editor: Close after timeout
    • In the Workflow Editor, Wait and Human Activity nodes can now automatically close after a specified timeout.
    • The timeout can be defined using a selected Time Unit or a Custom Expression.
    • The close-on event for the Wait node is now optional; users must select at least one option: Close on event or Close after timeout.
    • A new property, "HasExpired", is available in expressions for subsequent nodes created after a Wait or Human Activity, indicating whether the activity closed due to timeout.
    (RDODCT-2909)
