LifeTime Management Console 11.29.0

Released on Jun 19, 2026

New in LifeTime Management Console 11.29.0

  • It is now possible to use service account tokens with a 24-hour overlap period when rotating authentication tokens. When a new token is generated for a service account, the previous token remains valid for 24 hours, allowing integrations to be updated without service interruptions. For this overlap to also apply to Performance Monitoring API calls, the target environment must be running Platform Server 11.42.0 or later.

    (R11CT-4861)

  • You can now create LifeTime service accounts that are scoped to a specific ODC organization. This is the recommended approach for scenarios where ODC needs to interact with O11 infrastructures, for example, data, logic interoperability, or application conversion. Organization-scoped service accounts prevent ODC-cross-tenant token reuse, so credentials issued for one ODC organization cannot be used by another.

    (RDUCO-343)
