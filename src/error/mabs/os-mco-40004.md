# OS-MCO-40004

## Error message

`Extensibility Configurations of <AssetName> do not follow a supported schema: <Detail>.`

## Cause

Your mobile app's or library's extensibility configurations don't follow the supported schema. The JSON structure doesn't conform to either the Universal Extensibility or Cordova-based Extensibility format.

## Impact

The mobile compilation fails, causing the native mobile build to fail.

## Recommended action

Update the extensibility configurations to follow either the [Universal extensibility configuration JSON schema](https://success.outsystems.com/documentation/outsystems_developer_cloud/building_apps/mobile_apps/configuring_mobile_apps/universal_extensibility_configurations_json_schema/) or the [Cordova-based extensibility configuration JSON schema](https://success.outsystems.com/documentation/outsystems_developer_cloud/building_apps/mobile_apps/configuring_mobile_apps/cordova_based_extensibility_configuration_json_schema/). Start by reviewing the details provided in the error message.

The Capacitor apps use only the Universal extensibility configuration JSON schema.

If the problem persists, create a case with [OutSystems Support](https://www.outsystems.com/support/portal/open-support-case?ErrorCode=OS-MCO-40004).
