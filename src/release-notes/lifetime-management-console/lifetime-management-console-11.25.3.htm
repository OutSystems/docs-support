LifeTime Management Console 11.25.3

Released on Jan 03, 2025

New in LifeTime Management Console 11.25.3

  • Improved LifeTimeAPI Swagger documentation to include information about permissions for using the API. (R11CT-1808)
  • Expanded the output of the following LifeTime APIs - GET /environments/{EnvironmentKey}/applications/{ApplicationKey}/, GET /environments/{EnvironmentKey}/applications/ and GET /applications/{ApplicationKey}/ to include the application version label and GET /applications/{ApplicationKey}/versions/ to include the environment key. (R11CT-3486)
  • Added a new environment-level setting that controls if deployment plans can be created in parallel. Removed the setting to disable executing deployment plans in parallel, since it was redundant with the parallel execution limit, which can be used for the same purpose. (R11CT-3680)

Bug Fixing

  • Fixed an issue while installing sample data for LifeTimeSDK on Oracle installations when there were many deployment plans to be exported. The exception raised would mention "ORA-00913 Too many values". (R11CT-3621)
  • The LifeTimeAPI Swagger documentation has been updated to clarify the behavior of the 'application version creation' process. It now specifies that the application version will only become available after the required processing is completed. (R11CT-3651)
  • Previously, when accessing the details of a user in LifeTime, the audit message could mistakenly suggest that the user was blocked in the environments they have access to. Now, the audit message indicates that information about the user's block status is being retrieved. (R11CT-371)
  • Fixed a problem where applications created after a deployment plan had been created could not be added to the plan because they would not be visible in the add application popup window. (RPM-1107) Application Lifecycle LifeTime
  • Fixed an issue where Tag&Deploy would assign a tag lower than the highest application version in the infrastructure. Now, Tag&Deploy will tag the application in a way that respects the highest version present in the infrastructure. (RPM-4870) Application Lifecycle LifeTime
  • Improved audit logs collected during staging process by removing records that were generated unnecessarily. (RPM-5280) Application Lifecycle Service Center
  • Fixed an issue that would prevent the cleanup of some old data records from past LifeTime processes. The issue had no functional impact to the operation of LifeTime. (RPM-5365) Application Lifecycle LifeTime
