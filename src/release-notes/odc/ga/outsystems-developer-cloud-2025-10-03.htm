OutSystems Developer Cloud 2025-10-03

Released on Oct 03, 2025

New

  • [Mentor AppGen] List Screen Composition
    • Mentor now intelligently selects the optimal UI pattern for lists based on data volume and type, offering a diverse range of options including tables, card lists, maps, galleries, and master-detail views.
      • These patterns can be explicitly defined in prompts or embedded within requirement documents, ensuring generated applications deliver list UIs that are functionally accurate, navigable, and aligned with OutSystems UX guidelines.
    Sample Data Enhancements
    • Expanded the categories of data that have sample images.
    • Increased sample data records to 13 to add pagination for galleries.
    • Added a default image when ImageURL is empty, ensuring consistent layout.
    Visual Enhancements
    • App Menu: App Icon styling reviewed to create contrast with Menu color.
    • Dashboards:
      • Charts:
        • Pie and Donut charts now feature legends positioned alongside the chart.
        • Bar and Column charts have been updated with rounded borders.
      • Counters: The secondary text size in counters has been reduced for improved readability.
      • Layout: Headings and padding between screen elements have been reviewed and adjusted for a more consistent look.
    • Master Detail Screen:
      • Screen styling has been enhanced, improving the alignment of the card with the table height and the pagination component's position relative to the table.
      • Flow actions in Master Details are now created inside of the card area.
    • List Screen:
      • The filters component has been improved to display items sorted, without cropping content or breaking the layout.
      • To ensure consistency, Boolean check icons have been replaced with "Yes/No" indicators.
      • Tables' columns now follow the standard alignment used in ODC Studio.
      • Styling reviewed to apply the primary color to buttons.
    • View Screen: Text Handling - Content is always displayed below the label, creating consistency among data types.
    (RDMAG-871)

Bug Fixing

  • [Mentor AppGen] App Generation crashes:
    • Fixed generation crashes related to Invalid app name and authorization expressions.
    Fixed Generated App Inconsistencies:
    • Popup wasn't selected as the screen type.
    • Inconsistencies when referencing external entities.
    • Stateflows generated around static entities that didn't reflect statuses.
    • The user entity is only used as the TargetEntity for another entity identifier.
    • Invalid OML when the User entity is only used as the TargetEntity for another entity identifier.
    • Removed Unused Icon Resources from generated Apps.
    • Dashboards
      • Adding an Entity breaks the dashboard counters.
      • Dashboards showing lists of all records, which are not suitable for that screen.
      • Applications without dashboards, even when requested.
    • List Screen
      • For the card list, just the value should have a link.
    • Login Screen
      • Typo error in the Login screen.
    • Vertical Menu
      • Invalid Permission screen doesn't have the Logout option.
    Fixed Sample Data Issues:
    • Wrong Foreign Keys (FKs), creating apps without sample data.
    • Runtime Error when Extended Entity has fewer than 20 records.
    • Sample data wasn't always being generated, or some entities lacked sample data.
    • Some entities had the wrong data types.
    (RDMAG-875)
  • App Editor
    • Fixed the Infinite cycle message in the sitemap when opening the app
    • Fixed an issue with the Orphan screen after deleting a List Screen
    • Fixed the issue where some screens were disconnected from the sitemap
    (RDMOR-2665)
