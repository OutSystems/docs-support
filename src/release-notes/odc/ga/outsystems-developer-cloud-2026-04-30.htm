OutSystems Developer Cloud 2026-04-30

Released on Apr 30, 2026

New

  • Workflow Editor: Publish with message
    • In the Workflow Editor, it's now possible to publish a new revision of a workflow, with a message.
    • This feature lets users attach an optional message to a publish operation in the Workflow Editor.
    • The message is stored with the revision and is displayed in the revision history timeline, in publish-conflict dialogs, and in the open-workflow conflict dialog, giving users a clear audit trail of what each revision contained.
    (RDODCT-4220)
  • Monthly Security Update (April 2026): Updated application container base image, including security fixes. Check the Microsoft .NET 8.0.26 release for more details: https://github.com/dotnet/core/blob/main/release-notes/8.0/8.0.26/8.0.126.md (RRCT-7009)

Bug Fixing

  • Fix keyboard accessibility issue on Button Group Widget. (RPM-6337)
  • Content-Length header value is no longer ignored when explicitly set in the OnBeforeRequest callback of a Rest Consume output. Content-Length is not present in the input parameter. When it is not present in the output parameter, the value will be automatically calculated based on text/binary size. (RRCT-6548)
  • HTTPMethod value is no longer ignored when explicitly set in the OnBeforeRequest callback of a Rest Consume output. (RRCT-6550)
  • Content-Type header is now present in the list of headers in the OnBeforeRequest input parameter. (RRCT-6553)
  • Rest Consume URL Path is no longer partially encoded in runtime and is now consistent with the expected behavior developers set and test in ODC Studio. (RRCT-6556)
  • Content-Type header defined in Rest Consume HTTP Headers and input parameters no longer enforces the UTF-8 charset. The value defined by the developer will be used without any transformation. (RRCT-6559)
  • URL query parameters are now consistently encoded when compared with path parameters, and the plus character is avoided to prevent ambiguous encoding. (RRCT-6927)
