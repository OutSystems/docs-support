# OS-MCO-40011

## Error message

`Value for AppShield configuration <ConfigName> is not a valid <ConfigType>.`

## Cause

The AppShield configuration value doesn't match the expected data type for that configuration.

## Impact

The mobile compilation fails, causing the native mobile build to fail.

## Recommended action

Find the AppShield configuration mentioned in the error message and update its value to match the expected type. You'll most likely find it in your mobile app's extensibility settings or extensibility configurations.

Here's an example of a value for each type:

| Type                     | Example Value                                                    |
| ------------------------ | ---------------------------------------------------------------- |
| Base64                   | RXhhbXBsZQo=                                                     |
| Boolean                  | true                                                             |
| 256bit hexadecimal value | 0123456789ABCDEF0123456789ABCDEF0123456789ABCDEF0123456789ABCDEF |
| URL                      | <https://www.example.com>                                        |

If the problem persists, create a case with [OutSystems Support](https://www.outsystems.com/support/portal/open-support-case?ErrorCode=OS-MCO-40011).
