Platform Server 11.41.0

Released on Mar 23, 2026

New in Platform Server 11.41.0

  • Added underlying infrastructure capabilities to support O11 to ODC Data Interoperability, specifically for some system entities. (R11BCT-194)
  • Updated Oracle Managed Driver to version 23.26.100. See the Oracle release notes, for more information on related fixes. (R11BCT-321)
  • Deploy Service is now using the new primary communication port (12101) introduced in PS 11.24.0. The new primary port is necessary for product evolution and technology resiliency and its value must be distinct from the secondary port. (R11BRT-2275)
  • Updated Swagger UI dependency js-yaml to version 4.1.1. This update has no runtime impact. (R11DT-3581)
  • Updated Swagger UI dependency lodash to version 4.17.23. This update has no runtime impact. (R11DT-3616)

Bug Fixing

  • Fixed an issue in Monitoring screens that allow "errors only" filter to work properly. Suggestion made by Evert . (R11CT-3514) Application Lifecycle Service Center
  • Fixed an issue where the "Active with Errors" filter failed to update correctly. This occurred in the Process Details page, specifically when a process instance with the "Active with Errors" status had its error skipped. Although the instance's status changed to "Active", it remained visible in the "Active with Errors" filtered list. Suggestion made by Kilian. (R11CT-3515) Application Lifecycle Service Center
  • Fixed an issue within Traditional applications where an incomplete surrogate pair in an input field led to application unresponsiveness. (RPM-1709) Application Runtime Interface
  • Fixed an issue where "If" Widgets within emails would fail to render styles or evaluate expressions correctly in some scenarios, leading to unstyled content or missing data in the final email output. (RPM-4660) Application Runtime Interface
  • Fixed an issue in Service Center Extension logs page where the exported logs didn't match the logs shown on the screen. This would only happen if Log Separation was active and no specific extension was selected in the dropdown filter. (RPM-6481) Application Runtime Logging
  • Fixed an issue in Service Center Extension logs page when clicking the "Next" button triggered an error message indicating an invalid filter operator. This would only happen if Log Separation was active. (RPM-6520) Application Runtime Logging
  • Fixed an issue in Service Center Extension logs page which triggered an error message indicating an inline attribute placeholder was already initialized. This would only happen if Log Separation was active and an invalid date filter was used. (RPM-6522) Application Runtime Logging
  • It is now possible to see a log in Service Center when deleting modules, extensions and apps. These new logs will contain the User performing the action, and the components being deleted when applicable. (RPM-870) Application Lifecycle Service Center
