LifeTime Management Console 11.27.0

Released on Jun 18, 2025

New in LifeTime Management Console 11.27.0

  • By using the new feature Deployment Protection Policy, it is now possible to block an app from being deployed to specific environments. This prevents accidental deployments to incorrect environments. It is also possible to use LifeTime APIs to block and unblock an app from deploying to specific environments. An infrastructure-level setting can be used to define if new apps are blocked by default. (R11CT-3955)
  • Stagings can now be relabeled. (R11CT-4070)

Bug Fixing

  • Fixed an issue where the 'Staging Settings' screen was being skipped when deploying an application containing 'Entities from External Databases' for the first time to the target environment. This skip was leading to errors in the deployment of such applications. (RPM-5761) Application Lifecycle LifeTime

Known issues

  • On the Deployment Plans list screen, when a plan is saved but not yet started, clicking the plan label incorrectly redirects to the Staging_Progress screen. As a workaround, manually change the screen URL by replacing 'Staging_Progress' with 'Staging_Edit', then proceed with the plan as usual. This issue was fixed in LifeTime 11.27.1.
